## 4. 已发现的问题

### 4.1 NetCDF/HDF 问题

原先 15GB SNAP NetCDF 文件可以被 `ncinfo` 看到变量目录，但全分辨率变量读取失败。

失败变量：

```text
Sigma0_VV
incident
lat
lon
```

失败信息包括：

```text
HDF error (NC_EHDFERR)
H5D__btree_decode_key bad coordinate offset
```

已测试方法：

1. MATLAB `ncread` 小块读取；
2. MATLAB `h5read` 小块读取；
3. MATLAB 底层 `netcdf.getVar`；
4. Python `h5py` 读取。

结论：即使只读取 `1×1` 像元也失败，因此不是普通内存不足，而是高分辨率数据集的 HDF5 chunk 索引/偏移异常或版本兼容问题。

### 4.2 CF NetCDF 导出问题

SNAP 导出的 `NetCDF4-CF` 文件为无效文件：

```text
文件头全 0
MATLAB ncinfo: Unknown file format
```

结论：当前环境不再使用 CF NetCDF 作为 SAR 主数据格式。

### 4.3 `incident` 虚拟波段问题

`incident` 在 `.dim` 中仍显示为虚拟波段：

```text
<VIRTUAL_BAND>true</VIRTUAL_BAND>
<EXPRESSION>incident_angle</EXPRESSION>
```

但这不影响后续处理，因为 MATLAB 可直接读取：

```text
tie_point_grids/incident_angle.img
```

后续在代码中将 6×21 tie-point 入射角插值到 25×25 抽样点即可。

## 5. 当前决策

SAR 主输入不再使用 NetCDF，而使用 BEAM-DIMAP：

```text
S1A_GulfMexico_20260610_Cal_Deb.dim/.data
```

主要原因：

1. NetCDF/HDF 高分辨率变量读取失败；
2. CF NetCDF 导出无效；
3. BEAM-DIMAP 的 ENVI `.img/.hdr` 已验证可稳定读取；
4. 数据结构足够支撑 CMOD5.N 反演所需变量。
