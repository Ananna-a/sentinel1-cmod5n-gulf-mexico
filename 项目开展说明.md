## 3. 各软件的协作关系

### 3.1 SNAP

SNAP 是 ESA 提供的 Sentinel 数据处理软件。本项目中它主要负责把原始 SAR 产品预处理为 MATLAB 可稳定读取的 BEAM-DIMAP 数据。

对 Sentinel-1 SLC 数据，常见处理链为：

1. 打开 `.zip` 原始产品；
2. Apply Orbit File，更新精密轨道；

4. Radiometric Calibration，得到 `Sigma0_VV`；
5. TOPSAR Deburst，将 IW 子波束拼接为连续图像；
6. 视情况做 Speckle Filter 或 Terrain Correction；
7. Save Product As 为 BEAM-DIMAP，供 MATLAB 读取 `.img/.hdr`。

## 4. CMOD5.N 模型原理

### 4.1 SAR 为什么能反演海面风速

Sentinel-1 SAR 是主动微波雷达，会向海面发射 C 波段微波并接收回波。海面风越大，通常会产生越强的短重力毛细波，使海面粗糙度增大，雷达后向散射系数 `sigma0` 也随之变化。

但是 `sigma0` 不只由风速决定，还与以下因素有关：

1. 雷达入射角 `theta`；
2. 风向相对于雷达视线方向的夹角 `phi`；
3. 雷达极化方式，本项目使用 VV 极化；
4. 海面状态、降雨、浪场、近岸影响等。

因此需要地球物理模型函数 GMF。CMOD5.N 就是 C 波段 VV 极化 SAR/散射计常用的海面风场经验模型。

### 4.2 CMOD5.N 正向模型

CMOD5.N 的正向形式可以概括为：

```text
sigma0 = B0(V, theta) * [1 + B1(V, theta) * cos(phi) + B2(V, theta) * cos(2phi)]^p
```

其中：

| 符号 | 含义 |
|---|---|
| `sigma0` | 雷达归一化后向散射系数，必须使用线性单位 |
| `V` | 10 m 海面风速，单位 m/s |
| `theta` | SAR 入射角 |
| `phi` | 相对风向，即风向与雷达观测方位之间的夹角 |
| `B0` | 各向同性项，主要描述风速和入射角对散射强度的基础影响 |
| `B1` | 一阶谐波项，描述上风/下风不对称 |
| `B2` | 二阶谐波项，描述顺风/逆风和侧风差异 |
| `p` | 指数项，参考代码中使用 1.6 |

参考代码中把 `B0/B1/B2` 写成由 26 个经验系数控制的函数。这些系数来自 CMOD5.N 模型，不需要自己拟合。

### 4.3 为什么要“反演”

CMOD5.N 本质上是正向模型：

```text
给定 风速 V、入射角 theta、相对风向 phi
计算 理论后向散射 sigma0_model
```

但本作业的目标相反：

```text
已知 SAR 观测 sigma0_obs、入射角 theta、ERA5 风向 phi
求 风速 V
```

所以参考代码使用遍历法：

1. 假设风速从 0.1 m/s 到 30 m/s；
2. 每隔 0.05 m/s 取一个候选风速；
3. 对每个候选风速计算 `sigma0_model`；
4. 找到 `abs(sigma0_model - sigma0_obs)` 最小的风速；
5. 该风速就是当前网格点的 SAR 反演风速。

对应数学表达：

```text
V_retrieved = argmin_V | sigma0_model(V, theta, phi) - sigma0_obs |
```

### 4.4 单位问题

`Sigma0_VV` 有两种常见单位：

1. 线性单位：例如 0.01、0.05；
2. dB 单位：例如 -20 dB、-13 dB。

CMOD5.N 公式中使用线性 `sigma0`。如果输入是 dB，需要转换：

```text
sigma0_linear = 10^(sigma0_dB / 10)
```

参考代码中的逻辑是：

```matlab
if any(sig_db(:) < 0)
    sig_linear = 10.^(sig_db / 10);
else
    sig_linear = sig_db;
end
```

这个判断适合课堂作业，但正式处理时最好在 SNAP 或 MATLAB 中确认 `Sigma0_VV` 的单位，因为线性 `sigma0` 也可能存在异常负值或无效值。

### 4.5 风向问题

CMOD5.N 需要的是“相对风向 `phi`”，不是简单的地理风向。严格来说：

```text
phi = 风向 - 雷达观测方位角
```

参考脚本目前直接把 ERA5 风向 `dir_era5` 当作 `phi` 使用，这是一个简化处理，能跑通课堂作业流程，但物理上不够严谨。后续如果想让结果更可信，应尝试从 Sentinel-1 元数据或 SNAP 导出信息中获得雷达视向/方位角，再计算相对风向。

代码逻辑：

1. 读 `lat/lon/incident_angle/Sigma0_VV`；
2. 如果入射角不是全分辨率，插值到 SAR 图像尺寸；
3. 将整幅 SAR 图像均分为 25×25 个子块；
4. 每个子块取中心点；
5. 保存 `.dat` 文件和 `SAR_info_25x25.mat`。

建议修改：

1. `SAR_dir = 'zone2_20250506'` 改成你自己的实验名；
2. `file_path = 'process_data.nc'` 改成实际 SAR nc 路径；

4. 15 GB 的全分辨率 nc 很大，MATLAB 一次读取全图可能内存压力较大，后续可以改成按中心点索引局部读取。

### 5.2 第二步：`ERA5_read_match.m`

功能：将 ERA5 风场插值到 SAR 的 25×25 网格。

ERA5 风速计算：

```text
wind_speed = sqrt(u10^2 + v10^2)
```

气象风向计算：

```text
wind_dir = 180 + atan2(u10, v10) * 180 / pi
wind_dir = mod(wind_dir, 360)
```

这里的风向表示“风从哪里来”，以正北为 0°，顺时针增加。

建议修改：

1. `era5_filename` 改成 `数据/era5_wind_20260611.nc` 或复制到工作目录；
2. 检查 ERA5 维度顺序，当前文件是 `u10(time, latitude, longitude)`，而参考 MATLAB 代码假设 `u10_raw(:,:,1)`，可能需要根据 MATLAB `ncdisp` 结果调整；
3. 如果 `interp2` 报纬度不是递增，需要翻转 ERA5 纬度和对应风场矩阵。

### 5.3 第三步：`Inverse.m`

功能：用 CMOD5.N 遍历反演风速。

输入：

| 文件 | 变量 |
|---|---|
| `inc.dat` | 25×25 入射角 |
| `sig.dat` | 25×25 观测后向散射 |
| `dir.dat` | 25×25 ERA5 风向 |

输出：

| 文件 | 含义 |
|---|---|
| `sar_wind_speed.dat` | SAR 反演风速 |
| `SAR_info_25x25.mat` | 增加 `sar_wind_speed` |

核心逻辑：

```matlab
for 每个格点
    for v = 0.1:0.05:30
        计算 CMOD5.N 理论 sigma0
        计算理论值与观测值残差
    end
    取残差最小时的 v 作为反演风速
end
```

建议修改：

1. 加入 `NaN` 和异常 `sigma0` 过滤；
2. 保存最小残差矩阵，便于报告说明样本筛选；
3. 后续可把遍历法改为 `fminbnd` 优化，但课堂作业用遍历法更直观。

### 5.4 第四步：`Check.m`

功能：将 SAR 反演风速与 ERA5 风速对比。

指标：

```text
RMSE = sqrt(mean((SAR - ERA5)^2))
BIAS = mean(SAR - ERA5)
R = corr(SAR, ERA5)
MAE = mean(abs(SAR - ERA5))
```

参考脚本会先剔除 `0/NaN`，再剔除超过 `1.5 * RMSE` 的异常点，然后计算最终指标。

建议报告中同时说明：

1. 原始总点数是 625；
2. 有效点数是多少；
3. 是否进行了异常点剔除；
4. RMSE、相关系数、偏差分别说明什么。

### 5.5 第五步：`Toexcel.m`

功能：导出最终结果表。

输出字段：

| 字段 | 含义 |
|---|---|
| `Longitude` | 经度 |
| `Latitude` | 纬度 |

### 7.1 第一章：Sentinel-1 SAR 卫星数据下载

6. SNAP 预处理流程；
7. 导出变量：`Sigma0_VV`、经纬度、入射角。

### 7.2 第二章：CMOD5.N 海面风速反演及验证

建议结构：

1. 说明 SAR 后向散射与海面风速的物理关系；
2. 写出 CMOD5.N 基本公式；
3. 说明反演方法：固定入射角和风向，遍历风速求最小残差；
4. 说明 ERA5 匹配方法：U/V 分量转风速风向，双线性插值到 SAR 网格；
5. 给出 RMSE、相关系数、BIAS、MAE；
6. 插入折线对比图和散点图。

### 7.3 第三章：海面风速分布图绘制

如果做 Surfer 图，可以写：

1. 将 25×25 格点导出为 Excel；
2. 以经纬度为坐标、反演风速为 Z 值进行网格化；
3. 绘制等值线图或色斑图；
4. 分析高风速/低风速区域的空间位置；
5. 对比 ERA5 风速分布，说明差异来源。

### 7.4 第四章：总结与体会

可以从以下角度写：

1. SAR 能提供比 ERA5 更高空间分辨率的海面风速信息；
2. CMOD5.N 依赖准确的后向散射、入射角和相对风向；
3. ERA5 适合作为验证基准，但空间分辨率较粗；
4. 误差可能来自时间匹配、风向简化、近岸污染、SAR 噪声、模型适用范围等；
5. 后续可改进相对风向计算和异常样本筛选。
